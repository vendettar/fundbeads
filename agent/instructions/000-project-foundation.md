# 000 Project Foundation

## Goal

Create the initial Fundbeads project architecture by adapting the structural conventions of SqlSight without copying SqlSight product logic or database/backend assumptions.

## Scope

- Client-only Vite React app.
- React and TypeScript frontend under `frontend/`.
- Tailwind CSS v4 styling and generated design tokens from `DESIGN.md`.
- pnpm workspace scripts from the repository root.
- Docker static hosting with no backend service.
- `docs/`, `agent/instructions/`, and `agent/role-prompt/` workflow folders.

## Required Changes

- Keep the root workspace minimal: `package.json`, `pnpm-workspace.yaml`, Docker files, design generation script, docs, agent files, and `frontend/`.
- Ensure `frontend/src/design-theme.generated.css` is generated by `scripts/generate-design-theme.mjs`.
- Ensure the production image serves only compiled static assets.
- Document the client-only runtime model and local setup commands.
- Keep project docs aligned with current implementation, not planned future capability.

## Forbidden

- Do not add a backend, API server, database, server-side image storage, or image upload service.
- Do not copy SqlSight database, API, filesystem, SQLite, or Go roles into Fundbeads.
- Do not hand-edit generated design CSS.

## Acceptance Criteria

- `pnpm install` succeeds.
- `pnpm design:generate` creates the frontend design theme CSS.
- `pnpm check` builds and tests the frontend.
- The app can upload an image, generate a bead pattern, and show color counts locally.
- The docs explain how to run, build, preview, and deploy the static app.
- Agent role prompts describe Fundbeads risks rather than SqlSight risks.

## Required Tests

- Frontend unit tests for pattern utilities must pass.
- Build validation must pass through `pnpm check`.

## Verification Commands

```sh
pnpm design:generate
pnpm check
```

## Decision Log

Required for durable architecture changes. Waived for simple documentation or prompt wording updates that do not change the runtime model.
